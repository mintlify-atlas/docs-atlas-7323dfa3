title: "Role Management"
description: "List available roles, assign roles to users, and remove role assignments. Accessible to PLATFORM_ADMIN and COMPANY_ADMIN."
---

All endpoints require a valid JWT access token:

```bash
Authorization: Bearer <your_access_token>
```

<Info>
  Only `PLATFORM_ADMIN` and `COMPANY_ADMIN` can access role management endpoints. A `COMPANY_ADMIN` can only assign or remove roles within their own company.
</Info>

---

## Roles overview

The platform uses four roles. Each role may optionally be scoped to a company.

| Role | Description | Requires company |
|------|-------------|-----------------|
| `USER` | Standard end user — can create and track tickets | No |
| `AGENT` | Support agent — handles tickets for a specific company | Yes |
| `COMPANY_ADMIN` | Company administrator — manages their company, agents, and announcements | Yes |
| `PLATFORM_ADMIN` | Platform-level administrator — full system access | No |

---

## List roles

Returns all roles available in the system.

```bash
GET /api/roles
```

**Example**

```bash
curl --request GET \
  --url https://your-helpdesk.com/api/roles \
  --header 'Authorization: Bearer <token>'
```

## Assign role

<Warning>
  Throttled to **100 requests per 60 minutes** per authenticated user. Exceeding this returns `429 Too Many Requests`.
</Warning>

Assigns a role to a user. If the user previously had this role and it was deactivated, the assignment is **reactivated** and the response returns `200`. A brand-new assignment returns `201`.

```bash
POST /api/users/{userId}/roles
```

**Path parameters**

<ParamField path="userId" type="string" required>
  UUID of the user to assign the role to.
</ParamField>

**Request body**

<ParamField body="roleCode" type="string" required>
  Role to assign. One of: `USER`, `AGENT`, `COMPANY_ADMIN`, `PLATFORM_ADMIN`.
</ParamField>

<ParamField body="companyId" type="string">
  UUID of the company to scope the role to.

  - **Required** when `roleCode` is `AGENT` or `COMPANY_ADMIN`.
  - **Must be `null` or omitted** when `roleCode` is `USER` or `PLATFORM_ADMIN`.
</ParamField>

<Note>
  Providing `companyId` for `USER` or `PLATFORM_ADMIN`, or omitting it for `AGENT` or `COMPANY_ADMIN`, will cause a `422 Unprocessable Entity` validation error.
</Note>

**Example — assign AGENT role**

```bash
curl --request POST \
  --url https://your-helpdesk.com/api/users/9d4e8c91-5f2a-4b3e-8a1f-2c3d4e5f6a7b/roles \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "roleCode": "AGENT",
    "companyId": "8c3d5e1f-2a4b-5c6d-7e8f-9a0b1c2d3e4f"
  }'
```

**Response `201` — newly assigned**

```json
{
  "success": true,
  "message": "Role assigned successfully",
  "data": {
    "id": "7b5a4c3d-9e8f-1a2b-3c4d-5e6f7a8b9c0d",
    "roleCode": "AGENT",
    "roleName": "Agent",
    "company": {
      "id": "8c3d5e1f-2a4b-5c6d-7e8f-9a0b1c2d3e4f",
      "name": "Acme Corporation",
      "logoUrl": "https://your-helpdesk.com/storage/logos/acme.png"
    },
    "isActive": true,
    "assignedAt": "2025-11-01T14:30:00Z",
    "assignedBy": {
      "id": "6a4b3c2d-8e7f-9a0b-1c2d-3e4f5a6b7c8d",
      "userCode": "USR-ADMIN-001",
      "email": "admin@your-helpdesk.com"
    }
  }
}
```

**Response `200` — reactivated**

The response body has the same shape as `201` but `message` will indicate the role was reactivated.

<ResponseField name="data" type="object">
  <Expandable title="role assignment properties">
    <ResponseField name="id" type="string">UUID of the `UserRole` assignment record. Use this ID when removing the role.</ResponseField>
    <ResponseField name="roleCode" type="string">Assigned role code.</ResponseField>
    <ResponseField name="roleName" type="string">Human-readable role name.</ResponseField>
    <ResponseField name="company" type="object | null">Company the role is scoped to. `null` for `USER` and `PLATFORM_ADMIN` roles.</ResponseField>
    <ResponseField name="isActive" type="boolean">Whether the assignment is currently active.</ResponseField>
    <ResponseField name="assignedAt" type="string">ISO 8601 timestamp of when the role was assigned.</ResponseField>
    <ResponseField name="assignedBy" type="object | null">The admin who made the assignment.</ResponseField>
  </Expandable>
</ResponseField>

## Remove role

Deactivates a role assignment (soft delete). The `UserRole` record is preserved for audit purposes.

```bash
DELETE /api/users/roles/{roleId}
```

<Warning>
  The `{roleId}` path parameter is the **UserRole assignment UUID** (the `id` field returned when assigning a role), **not** the Role UUID. Using the wrong UUID will result in a `404 Not Found`.
</Warning>

**Path parameters**

<ParamField path="roleId" type="string" required>
  UUID of the `UserRole` assignment record to deactivate.
</ParamField>

**Query parameters**

<ParamField query="reason" type="string">
  Optional reason for removing the role. Maximum 500 characters.
</ParamField>

**Authorization rules**

| Role | Scope |
|------|-------|
| `PLATFORM_ADMIN` | Can remove any role assignment |
| `COMPANY_ADMIN` | Can only remove roles from users within their own company |

**Example**

```bash
curl --request DELETE \
  --url 'https://your-helpdesk.com/api/users/roles/7b5a4c3d-9e8f-1a2b-3c4d-5e6f7a8b9c0d?reason=User+changed+departments' \
  --header 'Authorization: Bearer <token>'
```

**Response `200`**

```json
{
  "success": true,
  "message": "Role removed successfully"
}
```

## Error responses

```json
{
  "success": false,
  "message": "Error description",
  "code": "ERROR_CODE"
}
```

| HTTP status | When it occurs |
|-------------|---------------|
| `400` | Business logic validation failed |
| `401` | Missing or invalid JWT token |
| `403` | Authenticated but insufficient permissions |
| `404` | User or role assignment not found |
| `422` | Input validation failed — e.g. missing `companyId` for an `AGENT` role |
| `429` | Rate limit exceeded |
