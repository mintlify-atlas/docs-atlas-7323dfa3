---
title: "Users & Profiles"
description: "Manage user accounts, profiles, preferences, and avatars. Includes admin-level user listing and status management."
---

All endpoints require a valid JWT access token unless otherwise noted. Include it as a Bearer token in every request:

```bash
Authorization: Bearer <your_access_token>
```

<Info>
  **COMPANY_ADMIN scope** — A `COMPANY_ADMIN` can only see and act on users that belong to their own company. Attempting to access users from other companies returns `403 Forbidden`.
</Info>

## Current user

<Info>Requires authentication.</Info>

Returns the full profile, active role contexts, and activity statistics for the authenticated user.

```bash
GET /api/users/me
```

### Update profile

<Warning>
  Throttled to **30 requests per hour** per authenticated user. Exceeding this limit returns `429 Too Many Requests`.
</Warning>

Updates the authenticated user's profile. All fields are optional — only the fields you send will be updated.

```bash
PATCH /api/users/me/profile
```

**Request body**

<ParamField body="firstName" type="string">
  First name. 2–100 characters.
</ParamField>

<ParamField body="lastName" type="string">
  Last name. 2–100 characters.
</ParamField>

<ParamField body="phoneNumber" type="string">
  Phone number. 10–20 characters. Digits, spaces, `+`, `-`, `(`, `)` allowed.
</ParamField>

<ParamField body="avatarUrl" type="string">
  Avatar URL. Maximum 2048 characters.
</ParamField>

**Example**

```bash
curl --request PATCH \
  --url https://your-helpdesk.com/api/users/me/profile \

  --header 'Content-Type: application/json' \
  --data '{
    "firstName": "Jane",
    "lastName": "Smith",
    "phoneNumber": "+19876543210"
  }'
```

**Response `200`**

```json
{
  "data": {
```

### Update preferences

<Warning>
  Throttled to **50 requests per hour** per authenticated user.
</Warning>

Updates language, timezone, and notification preferences. All fields are optional.

```bash
PATCH /api/users/me/preferences
```

**Request body**

<ParamField body="theme" type="string">

</ParamField>

<ParamField body="language" type="string">

</ParamField>

<ParamField body="timezone" type="string">
  IANA timezone identifier, e.g. `America/New_York`.
</ParamField>

<ParamField body="pushWebNotifications" type="boolean">
  Enable or disable push web notifications.
</ParamField>

<ParamField body="notificationsTickets" type="boolean">

```bash
curl --request PATCH \

  --data '{
    "theme": "dark",
    "language": "en",
    "timezone": "America/Chicago",
    "pushWebNotifications": false,
    "notificationsTickets": true
  }'
```

**Response `200`**

### Upload avatar

<Warning>
  Throttled to **3 requests per hour** per authenticated user.
</Warning>

Uploads a new avatar image for the authenticated user. Send as `multipart/form-data`. Supported formats: JPEG, PNG, GIF, WebP. Maximum file size: **5 MB**.

```bash
POST /api/users/me/avatar
```

**Request body (multipart/form-data)**

<ParamField body="avatar" type="file" required>
  Image file. Accepted formats: `image/jpeg`, `image/png`, `image/gif`, `image/webp`. Max size: 5 MB.
</ParamField>

**Example**

```bash
curl --request POST \
  --url https://your-helpdesk.com/api/users/me/avatar \
  --header 'Authorization: Bearer <token>' \
  --form 'avatar=@/path/to/photo.jpg'
```

**Response `200`**

```json
{
  "message": "Avatar uploaded successfully",
  "data": {
    "avatarUrl": "https://your-helpdesk.com/storage/avatars/9d4e8c91.../1731774123_profile.jpg"
  }
}
```

## View a user

<Info>Requires authentication. Any user can view their own profile. Admins and agents can view users within their company.</Info>

Returns the full user record for the given UUID.

```bash
GET /api/users/{id}
```

**Path parameters**

<ParamField path="id" type="string" required>
  User UUID.
</ParamField>

**Authorization rules**

| Role | Can view |
|------|----------|
| Any authenticated user | Their own profile |
| `AGENT` | Users from their company |
| `COMPANY_ADMIN` | Users from their company |
| `PLATFORM_ADMIN` | Any user |

**Example**

```bash
curl --request GET \
  --url https://your-helpdesk.com/api/users/9d4e8c91-5f2a-4b3e-8a1f-2c3d4e5f6a7b \
  --header 'Authorization: Bearer <token>'
```

**Response `200`**

```json
{
  "data": {
    "id": "9d4e8c91-5f2a-4b3e-8a1f-2c3d4e5f6a7b",
    "userCode": "USR-20250001",
    "email": "jane.doe@example.com",
    "status": "ACTIVE",
    "emailVerified": true,
    "profile": { "..." : "..." },
    "roleContexts": [],
    "createdAt": "2025-01-15T10:30:00Z",
    "updatedAt": "2025-11-01T14:25:30Z"
  }
}
```

## Admin endpoints

<Note>
  The endpoints below require the `PLATFORM_ADMIN` or `COMPANY_ADMIN` role. A `COMPANY_ADMIN` automatically sees only users from their own company regardless of the filters applied.
</Note>

### List users

Returns a paginated, filterable list of users.

```bash
GET /api/users
```

**Query parameters**

<ParamField query="search" type="string">
  Case-insensitive search across email, user code, first name, and last name.
</ParamField>

<ParamField query="status" type="string">
  Filter by account status: `ACTIVE`, `INACTIVE`, or `SUSPENDED`.
</ParamField>

<ParamField query="emailVerified" type="boolean">
  Filter by email verification status.
</ParamField>

<ParamField query="role" type="string">
  Filter by role code: `USER`, `AGENT`, `COMPANY_ADMIN`, or `PLATFORM_ADMIN`.
</ParamField>

<ParamField query="companyId" type="string">
  Filter users who have an active role in the specified company UUID.
</ParamField>

<ParamField query="recentActivity" type="boolean">
  When `true`, returns only users active within the last 7 days.
</ParamField>

<ParamField query="createdAfter" type="string">
  ISO 8601 datetime. Returns users created after this timestamp.
</ParamField>

<ParamField query="createdBefore" type="string">
  ISO 8601 datetime. Returns users created before this timestamp.
</ParamField>

<ParamField query="order_by" type="string" default="created_at">
  Sort field. One of: `created_at`, `updated_at`, `email`, `status`, `last_login_at`, `last_activity_at`.
</ParamField>

<ParamField query="order_direction" type="string" default="desc">
  Sort direction: `asc` or `desc`.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number (1-indexed).
</ParamField>

<ParamField query="per_page" type="integer" default="15">
  Items per page. Maximum **50**.
</ParamField>

**Example**

```bash
curl --request GET \
  --url 'https://your-helpdesk.com/api/users?role=AGENT&status=ACTIVE&per_page=20' \
  --header 'Authorization: Bearer <token>'
```

**Response `200`**


### Update user status

<Warning>
  Requires `PLATFORM_ADMIN`. Only platform administrators can change user status.
</Warning>

Activates or suspends a user account.

```bash
PUT /api/users/{id}/status
```

**Path parameters**

<ParamField path="id" type="string" required>
  User UUID.
</ParamField>

**Request body**

<ParamField body="status" type="string" required>
  New account status: `active` or `suspended`.
</ParamField>

<ParamField body="reason" type="string">
  Required when `status` is `suspended`. Audit reason for the suspension.
</ParamField>

**Example**

```bash
curl --request PUT \
  --url https://your-helpdesk.com/api/users/9d4e8c91-5f2a-4b3e-8a1f-2c3d4e5f6a7b/status \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "status": "suspended",
    "reason": "Violation of acceptable use policy"
  }'
```

**Response `200`**

```json
{
  "data": {
    "userId": "9d4e8c91-5f2a-4b3e-8a1f-2c3d4e5f6a7b",
    "status": "suspended",
    "updatedAt": "2025-11-01T17:00:00Z"
  }
}
```

### Delete user

<Warning>
  Requires `PLATFORM_ADMIN`. This is a **soft delete** — the user record is preserved for audit purposes but the account is deactivated. You cannot delete your own account.
</Warning>

Soft-deletes a user (sets `status` to `deleted` and records `deleted_at`).

```bash
DELETE /api/users/{id}
```

**Path parameters**

<ParamField path="id" type="string" required>
  User UUID.
</ParamField>

**Query parameters**

<ParamField query="reason" type="string">
  Optional reason for deletion. Logged in the audit trail when the audit feature is available.
</ParamField>

**Example**

```bash
curl --request DELETE \
  --url 'https://your-helpdesk.com/api/users/9d4e8c91-5f2a-4b3e-8a1f-2c3d4e5f6a7b?reason=Account+closure+request' \
  --header 'Authorization: Bearer <token>'
```

**Response `200`**

```json
{
  "data": {
    "success": true
  }
}
```

## Error responses

All endpoints follow a consistent error format:

```json
{
  "success": false,
  "message": "Error description",
  "code": "ERROR_CODE"
}
```

| HTTP status | When it occurs |
|-------------|---------------|
| `401` | Missing or invalid JWT token |
| `403` | Authenticated but insufficient permissions |
| `404` | User not found |
| `422` | Validation failed — `errors` object contains field-level messages |
| `429` | Rate limit exceeded |
