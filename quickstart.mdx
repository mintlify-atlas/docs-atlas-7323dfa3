---
title: "Quick start"
description: "Get Helpdesk running locally with Docker in under five minutes."
---

<Note>
  Wait 30–60 seconds after starting the containers before running migrations. The PostgreSQL and Redis health checks need to pass first. Run `docker compose ps` to confirm all services show a healthy status.
</Note>

<Steps>
  <Step title="Clone the repo and start Docker">
    Clone the repository and bring up all services in detached mode.

    ```bash
    git clone https://github.com/Lukehowland/Helpdesk.git
    cd Helpdesk
    docker compose up -d
    ```

    Verify that every container is healthy before continuing:

    ```bash
    docker compose ps
    ```
  </Step>

  <Step title="Run migrations and seed the database">
    Apply all database migrations across the four PostgreSQL schemas and load the seed data.

    ```bash
    docker compose exec app php artisan migrate --seed
    ```
  </Step>

  <Step title="Optimize for performance">
    Cache config, routes, and views, and warm the OPcache.

    ```bash
    ./scripts/optimize-performance.sh
    ```
  </Step>

  <Step title="Register a user via the API">
    Create your first account. All fields are required.

    ```bash
    curl -X POST http://localhost:8000/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "email": "user@example.com",
        "password": "SecurePass123!",
        "passwordConfirmation": "SecurePass123!",
        "firstName": "John",
        "lastName": "Doe",
        "acceptsTerms": true,
        "acceptsPrivacyPolicy": true
      }'
    ```
  </Step>

  <Step title="Log in and get a JWT">
    Exchange your credentials for an access token. Copy the `accessToken` value from the response — you'll use it in the next step.

    ```bash
    curl -X POST http://localhost:8000/api/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "user@example.com",
        "password": "SecurePass123!"
      }'
    ```

    The response looks like:

    ```json
    {
      "success": true,
      "data": {
        "accessToken": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
        "refreshToken": "...",
        "expiresIn": 900
      },
      "message": "Login successful"
    }
    ```
  </Step>

  <Step title="Make your first authenticated request">
    Fetch the current user's profile using the Bearer token from the previous step.

    ```bash
    curl -X GET http://localhost:8000/api/users/me \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
    ```
  </Step>
</Steps>

## Access points

| Service | URL | Description |
|---|---|---|
| Application | http://localhost:8000 | Main Helpdesk web app and REST API |
| Mailpit | http://localhost:8025 | Email testing UI — catches all outbound mail |
| PostgreSQL | localhost:5432 | Database (user: `helpdesk`, db: `helpdesk`) |
| Redis | localhost:6379 | Cache and queue store |
